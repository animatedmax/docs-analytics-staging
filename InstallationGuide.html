Installation Guide

Overview of Installation

Installation of Pivotal AX requires working with your internal networking environment, a VMware vSphere ESXi environment, a Pivotal HD installation, and the Pivotal AX software. The Main Installation Steps table below describes the installation process and provides links to the procedures you should follow. -

When you install Pivotal AX, you first deploy a single virtual machine (VM) in a VMware vSphere environment. This single VM contains a pre-installed Pivotal Cloud Foundry  (CF) application called Pivotal CF Operations Manager that enables the installation of Pivotal AX and other CF services. You access this software using a Web browser with a network connection to this single VM.  Operations Manager allows you to describe server, Hadoop, networking, authentication, and other configuration parameters for your installation. The software then provisions and deploys multiple VMs in the vSphere environment based on the information you provide.

Before you begin installing Pivotal AX, make sure you have the required hardware and software available. See System Requirements and Sizing Guidelines.

3. Deploy hardware appropriate for your businessUnderstand the type of analysis you want to perform and the amount of data you need to analyze to determine the number and capacity of hardware and virtual machines required to run Pivotal AX.Sizing Guidelines
4. Prepare a vSphere ESX cluster for use with Pivotal AX

You deploy Pivotal AX on one or more VMware ESXi server(s). The size and number of ESX servers required depend on the amount of data and type of analysis you need to perform.

You use a vSphere client to configure your ESXi environment for use with Pivotal AX. See your VMware vSphere documentation for more information.

For help with determining the correct number and size of servers, see Sizing Guidelines.

VMware vSphere 5.5 Documentation Center
5. Deploy the Pivotal CF Operations Manager template as a virtual machine

You use a OVA template file to create a virtual machine that runs the Pivotal CF Operations Manager. You use Operations Manger to install Pivotal AX, and optionally, other Cloud Foundry services.

After you configure and start this virtual machine, the Pivotal CF Operations Manager application starts. You access this application using a Web browser that connects to services running on this virtual machine.

Deploying the Pivotal CF Operations Manager Virtual Machine
6. Run the Pivotal CF Operations Manager

Using the Pivotal CF Operations Manager Web application, you enter configuration values for Operations Manager.

Configuring Pivotal CF Operations Manager to Install Pivotal AX
7. Prepare a Pivotal HD installation for use with Pivotal AX

Pivotal AX requires a Pivotal HD (Hadoop) cluster. See your Pivotal HD documentation for installation and configuration information.

If your Pivotal HD deployment was installed using the Pivotal Command Center, see Preparing your Pivotal HD Deployment for Pivotal AX for further instructions.

If your Pivotal HD deployment was installed using the Pivotal CF Operations Manager, skip this step.

Preparing your Pivotal HD Deployment for Pivotal AX
8. Configure Pivotal AX

Using the Pivotal CF Operations Manager Web application, you enter configuration values for your Pivotal AX installation. Operations Manager uses this configuration information and connects to your vSphere cluster and provisions the virtual machines needed for your deployment, installs the Pivotal AX software, and starts the application. Depending on the size of your deployment this step may take up to several hours to complete and does not require user interaction.

Required Information for Installation

You will need the following information available before you begin the installation:

  • IP Address of the vSphere installation where you will deploy Pivotal AX. 
  • SMTP (outgoing mail server) connection information (IP address, port number, user name, and password)
  • Proxy server connection information (needed only if proxy servers are required in your network) 

  • Pivotal HD connection information—IP addresses and port numbers for the following Pivotal HD components:

    • HDFS namenode
    • Hbase master
    • Hive server
    • Jobtracker
    • Yarn resource manager
    • Yarn scheduler
    • Map Reduce History Server
    • Host mappings (host name and IP address) of all nodes in your Pivotal HD cluster
  • Networking information. See Networking Requirements.
  • HDFS endpoint connection information 
  • YARN resource manager connection information 
  • Hbase master connection information 
  • Name and email address of at least one administrative user for Pivotal AX
  • Email alias for user support
  • Email address to receive new user notifications
