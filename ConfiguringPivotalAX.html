Pivotal Product Documentation : Configuring Pivotal AX

This page last changed on Nov 22, 2013 by rawlie.

The following steps configure your Pivotal AX installation. After you fill in the necessary information, the Pivotal CF Installer creates the necessary virtual machines and installs the Pivotal AX software. Depending on the type and amount of hardware, network throughput, and the size of your installation, this process can take up to several hours to complete. 

Before beginning these steps, you must complete the vSphere, Pivotal CF Installer setup and your Pivotal HD cluster should be installed, configured, and running. See Installation Guide.

You should be logged on to the Pivotal CF Installer Web application to begin these steps. (See Configuring the Pivotal CF Installer to Install Pivotal AX for login instructions.)

  1. Click the Pivotal AX Box.
    The Pivotal AX configuration page displays.

  2. Select the Settings tab.
     

  3. Select Pivotal AX from the navigation menu on the left and enter the user name and password for your Pivotal AX Administrative User.

  4. Click Save.

  5. Select Network and Email from the left navigation menu and enter the following information about your company network:

  10. Click Save.
    Select Resource Sizes from the left menu.
    The Resource sizes configuration screen displays. On this screen, you define the number of instances applied to each of the resources required for your Pivotal AX installation. The number of instances for each job depend on the size of your Pivotal AX deployment and the type of analytics you intend to perform.

    For more information, see System Requirements and Sizing Guidelines.

    You can configure the number of instances, CPU, RAM, and disk space for each of the following types of nodes in your Pivotal AX deployment:

    • Zabbix and CF broker
    • Management
    • Messaging
    • Query
    • Storm
    • Compilation
